Accessible Links

Links are a part of almost every website on the internet. They take the user to different places, whether it is within the webpage or to another website. Making sure links can be accessed is essential since that is how we traverse through the web.

In this blog post I will explain the various pseudo-classes that apply to links. Styling these appropriately is the key to an accessible website.

First, we need to consider the various ways in which the user can access and navigate through web pages.

    Some ways users can access the web:

  • touch screen
  • mouse/trackpad
  • keyboard
  • single-switch input
  • mouth stick
  • head movement tracking
  • eye movement tracking
  • game controller
  • voice recognition
  • screen reader

While there are more, these can be narrowed down to three main inputs: mouse, touch, and keyboard. For example, a mouth stick, head movement tracking, or eye movement tracking acts as a mouse, since the cursor moves as the person moves. The single-switch input would usually be used with an on-screen keyboard, since they can navigate it and use it with a switch. Other ways to use an on-screen keyboard is with a mouse, touchscreen, or joystick. In each instance, basic navigation of either touch, mouse, or keyboard remains true.

With this in mind, the web developer will take care to style the basic pseudo classes of links in order for everyone to be able to navigate with ease.

    A link's pseudo-classes:

  • :link
  • :visited
  • :hover
  • :focus
  • :active

I have to say, only the first two pseudo-classes are specifically for links, while the others can apply to other HTML elements. They are important to include nonetheless. Let's go through each pseudo-class and learn what they control.

:link controls the style of an unvisited link- basically what it looks like when it is not being interacted with and hasn't been interacted with yet.

:visited controls the style of a link that has been visited! This lets the user know that they have already followed that particular link.

:hover controls the style of a link as it is being hovered over. This only applies to someone using a mouse or something that translates to a mouse to navigate the web.

:focus controls the style of a link as it is being focused on. When an element is in focus, it is being selected (not activated) by a keyboard. You can think of it as a hover state but for keyboard users.

:active controls the style of a link as it is being activated. This means as it is being touched, clicked, or activated by a keyboard. This state passes by very quickly, but applies to all inputs. It lets the user know that they are about to follow the link.

Including and styling each of these pseudo-classes not only increase accessibility, but overall user experience. Giving feedback to the user lets them know that they are interacting with the page, and consistency with this feedback will make the user comfortable with your site.
